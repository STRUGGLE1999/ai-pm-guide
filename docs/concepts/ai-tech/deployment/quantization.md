# CH6 大模型量化

量化是把模型参数从高精度表示转换成低精度表示，例如 FP32 到 FP16、INT8、INT4。它能降低显存和成本，但可能影响模型质量。

## 量化的价值

- 让更大模型跑在有限显存上。
- 降低推理成本。
- 提升吞吐。
- 支持边缘或私有化部署。

## 工程风险

量化后模型可能在复杂推理、格式遵从、专业术语上变差。因此量化不是只看服务能否启动，还要重新跑评测集。

## 检查

1. 量化前后质量差异是多少？
2. 哪些任务退化最明显？
3. 延迟和成本降低多少？
4. 是否有回滚方案？

量化是成本优化手段，不是无损压缩。

## 工程实践要点

部署与加速决定 AI 能力能否被真实用户持续使用。Demo 阶段只要能跑，生产阶段要看并发、延迟、成本、稳定性、安全和回滚。读者需要把“模型效果”扩展为“完整链路体验”。

| 指标 | 含义 | 工程影响 |
| --- | --- | --- |
| TTFT | 首个 Token 出现时间。 | 影响聊天等待感。 |
| 总延迟 | 完整任务完成时间。 | 影响用户是否愿意继续用。 |
| 吞吐 | 单位时间处理多少请求。 | 影响高峰期稳定性。 |
| 成本 | 单次任务消耗多少钱。 | 影响商业模式是否成立。 |
| 成功率 | 请求是否稳定完成。 | 影响信任和 SLA。 |

## 示例配置

下面是一个最小模型服务接口示例，用 FastAPI 暴露推理入口：

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class PredictRequest(BaseModel):
    text: str

@app.post("/predict")
def predict(req: PredictRequest):
    return {
        "label": "example",
        "score": 0.98,
        "input_length": len(req.text),
    }
```

真实部署需要补充模型加载、批处理、超时、日志、鉴权、限流、健康检查和版本管理。

## 技术细节拆解

大模型量化的核心，是用更低位宽保存权重或激活值，减少显存占用和内存带宽压力。它不是简单“压缩文件”，而是改变模型计算时的数值表示，所以必须同时看质量、速度、显存和退化样本。

| 层次 | 具体内容 |
| --- | --- |
| 输入 | 原始模型权重、校准数据、目标硬件、推理框架配置。 |
| 处理 | 选择量化位宽，统计数值范围，缩放/反量化，必要时做校准或量化感知训练。 |
| 输出 | INT8、INT4、FP16、BF16 等形式的模型权重或推理配置。 |
| 指标 | 显存占用、tokens/s、TTFT、P95 延迟、评测集分数、格式遵从率。 |

## 关键参数和边界

| 参数/边界 | 说明 |
| --- | --- |
| FP16 / BF16 | 常见半精度格式，速度和显存比 FP32 更友好，质量通常较稳。 |
| INT8 | 常用于较稳妥的推理量化，质量损失通常可控，但仍需评测。 |
| INT4 | 显存下降明显，但更容易损失推理、数学、格式遵从和专业术语能力。 |
| per-tensor / per-channel | per-channel 更细，通常更准，但实现和性能权衡更复杂。 |
| calibration data | 校准数据要接近真实输入，否则量化范围估计会偏。 |

## 可运行检查

```python
# 粗略估算不同精度下 70 亿参数模型的权重显存
params = 7_000_000_000
bytes_per_param = {
    "fp32": 4,
    "fp16/bf16": 2,
    "int8": 1,
    "int4": 0.5,
}

for name, b in bytes_per_param.items():
    gb = params * b / 1024 ** 3
    print(f"{name}: {gb:.2f} GiB")
```

这只是权重估算，真实推理还要加上 KV Cache、运行时 buffer、batch、上下文长度和框架开销。

## 怎么判断学懂了

| 判断点 | 具体标准 |
| --- | --- |
| 能估算显存 | 能区分权重显存、KV Cache 和运行时额外开销。 |
| 能解释退化 | 知道量化可能影响复杂推理、长文本、格式输出和专业术语。 |
| 能设计评测 | 量化前后用同一批样本比较质量、延迟、吞吐和失败类型。 |
| 能保留回滚 | 量化模型上线必须保留原精度或上一版本模型作为回退。 |

## 常见误区和排查

| 问题 | 为什么会发生 | 怎么排查 |
| --- | --- | --- |
| 量化后格式遵从变差 | 低位宽影响模型对固定 JSON、表格、代码格式的稳定输出。 | 用格式类评测集单独比较量化前后成功率。 |
| 显存估算过低 | 只算权重，没有算 KV Cache、batch 和运行时 buffer。 | 按上下文长度和并发重新估算峰值显存。 |
| 校准数据不匹配 | 校准样本和真实输入分布差距大。 | 用真实业务样本重新做校准和回归评测。 |

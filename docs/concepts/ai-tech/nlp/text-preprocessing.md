# 1.1 文本预处理

文本预处理是把原始文本变成模型更容易处理的输入。用户输入、网页正文、合同、客服对话、会议转写都很脏，直接丢给模型会带来噪声、截断、格式错乱和隐私风险。

## 常见处理步骤

| 步骤 | 做什么 | 工程影响 |
| --- | --- | --- |
| 清洗 | 去乱码、广告、页眉页脚、重复空格。 | 减少模型被无关内容干扰。 |
| 分句 / 分段 | 按语义拆开长文本。 | 影响摘要、RAG 切分和引用。 |
| Tokenization | 把文本切成模型单位。 | 影响成本、上下文长度和截断。 |
| 归一化 | 统一大小写、全半角、日期、数字。 | 提升匹配和抽取稳定性。 |
| 脱敏 | 隐去手机号、身份证、客户名等。 | 降低隐私和合规风险。 |

## 技术实现要点

预处理不是越多越好。合同、病历、政策文件里的格式、标题层级、表格和编号可能是关键证据，不能粗暴清掉。做知识库或文档问答时，预处理规则要和业务验证一起看。

## 常见坑

- OCR 后文字顺序错乱，导致模型理解错上下文。
- 切分太短，答案证据被拆散。
- 切分太长，召回片段噪声太多。
- 脱敏后关键字段丢失，影响抽取任务。

好的预处理应该让模型更容易找到证据，而不是把资料洗成看似干净但失去结构的文本。

## 不同文本来源的处理重点

| 来源 | 重点 | 风险 |
| --- | --- | --- |
| PDF / Word | 保留标题层级、表格、页码、附件关系。 | 解析后段落顺序错乱。 |
| 网页 | 去导航、广告、页脚，保留 URL 和更新时间。 | 把无关内容写入知识库。 |
| 聊天记录 | 保留说话人、时间、上下文。 | 断章取义或泄露隐私。 |
| OCR 文本 | 修正换行、识别错误、表格结构。 | 数字、姓名、金额识别错。 |
| 代码 / 日志 | 保留缩进、错误栈、时间戳。 | 格式被清洗破坏。 |

## 需要明确的处理规则

预处理规则最好不要只存在于某个脚本里。可以在解析规则文档中写清楚：

- 支持哪些文件类型。
- 每个文件最大多大。
- 是否保留表格和图片说明。
- 是否提取标题层级。
- 是否记录来源、版本、更新时间。
- 是否做敏感信息脱敏。
- 解析失败时怎么提示用户。

这些规则越清楚，后续 RAG、摘要、抽取、搜索的质量越容易稳定。

## 示例代码

下面用 Python 做一个最小文本预处理和词频统计：

```python
import re
from collections import Counter

text = "RAG uses retrieval, reranking, and generation. Retrieval quality matters."
tokens = re.findall(r"[a-zA-Z]+", text.lower())
counts = Counter(tokens)

print(tokens)
print(counts.most_common(5))
```

传统 NLP 和大模型应用都离不开文本清洗、切分、表示和评估。先用简单代码看清输入，再接复杂模型。

## 技术细节拆解

1.1 文本预处理 可以拆成输入、处理、输出和指标四层。这样读的时候不会停留在概念名，而是能看到它在系统里接收什么、改变什么、产出什么。

| 层次 | 具体内容 |
| --- | --- |
| 输入 | 原始文本、文档段落、OCR 结果、聊天记录、HTML 或日志。 |
| 处理 | 清洗、分句、切分、Tokenization、表示、分类/抽取/匹配。 |
| 输出 | 类别、实体、关系、摘要、相似度、检索片段或生成文本。 |
| 指标 | 精确率、召回率、F1、引用命中率、人工一致性。 |

## 关键参数和边界

| 参数/边界 | 说明 |
| --- | --- |
| 切分粒度 | 过短会丢上下文，过长会引入噪声并增加成本。 |
| 保留结构 | 标题、表格、列表、页码和说话人经常是重要语义。 |
| 实体边界 | 要定义清楚什么算实体，别名、缩写、编号都要处理。 |
| 脱敏规则 | 脱敏不能破坏关键字段，否则会影响抽取和匹配。 |

## 可运行检查

```python
import re

text = "张三在 2026-06-25 提交了合同，金额为 12000 元。"
dates = re.findall(r"\d{4}-\d{2}-\d{2}", text)
amounts = re.findall(r"\d+(?:\.\d+)?\s*元", text)
print({"dates": dates, "amounts": amounts})
```

## 怎么判断学懂了

| 判断点 | 具体标准 |
| --- | --- |
| 样本抽查 | 每次改清洗或切分规则后抽查原文、片段和输出。 |
| 长文本表现 | 单独评估长文档、表格、附件和 OCR 噪声。 |
| 证据链 | 抽取或生成结果要能追溯到原文位置。 |

## 常见误区和排查

| 问题 | 为什么会发生 | 怎么排查 |
| --- | --- | --- |
| 切分破坏语义 | 长文档被切断，答案证据分散。 | 抽查原文和 chunk，调整重叠窗口。 |
| 实体边界不一致 | 标注人对组织、区域、产品型号口径不同。 | 先写实体定义和反例。 |
| 清洗过度 | 标题、表格、编号被删除。 | 保留结构化元素并在解析后抽样检查。 |

# 对话式 AI

对话式 AI 是通过自然语言和用户持续交互的系统，包括客服机器人、销售助手、智能问答、企业助理、语音助手和 Agent 对话入口。它不只是把模型接到聊天框里，而是要处理意图、上下文、状态、知识、工具、转人工和安全边界。

## 对话系统的核心组成

| 组成 | 作用 |
| --- | --- |
| 意图识别 | 判断用户想做什么。 |
| 上下文管理 | 记住本轮和多轮对话里的关键信息。 |
| 知识检索 | 从知识库、文档、业务系统中找依据。 |
| 工具调用 | 查订单、改状态、创建工单、发通知。 |
| 回复生成 | 用合适语气输出答案或追问。 |
| 转人工 | 当模型不确定、权限不足或风险较高时交给人。 |

传统对话系统常用规则、意图分类和槽位填充；大模型对话系统更灵活，但也更需要评测、权限和兜底。

## 多轮对话的难点

用户经常省略信息，比如“那帮我改一下”“这个可以退吗”“刚才那个不要了”。模型必须知道“那”“这个”“刚才那个”指什么。

读者要设计清楚：

- 哪些信息需要记住？
- 记多久？
- 用户是否能查看和删除记忆？
- 哪些信息不能进入模型上下文？
- 信息不完整时模型是追问、拒答，还是转人工？

## 对话不是越像人越好

企业场景里，对话式 AI 更重要的是准确、可控、可追踪。过度拟人化可能让用户误以为系统有权限、能承诺结果、能代表公司做决定。

一个好的对话系统应该明确边界：

- 能回答什么。
- 能执行什么。
- 哪些需要人工确认。
- 哪些信息来自资料，哪些是推断。
- 失败时怎么办。

## 常见评测维度

| 维度 | 评估问题 |
| --- | --- |
| 任务完成率 | 用户是否完成查询、办理、生成或提交。 |
| 首轮解决率 | 不转人工能否解决高频问题。 |
| 追问质量 | 信息缺失时是否问得清楚。 |
| 知识 grounded | 是否基于资料回答并给出来源。 |
| 安全合规 | 是否拒绝越权、敏感、误导请求。 |
| 用户体验 | 是否简洁、礼貌、稳定，不绕圈。 |

## 工程实践要点

大模型应用的关键不是“接入模型”，而是把模型放进可控链路：用户输入、上下文、知识库、工具、权限、输出格式、人工确认和日志追踪。每个环节都可能决定功能能不能上线。

| 能力 | 适合解决 | 必须补上的约束 |
| --- | --- | --- |
| LLM | 理解、生成、总结、抽取。 | 输出格式、拒答、安全边界。 |
| RAG | 接企业知识和最新资料。 | 来源、权限、切分、召回和引用。 |
| Tool Calling | 查询或修改业务系统。 | 服务端鉴权、参数校验、二次确认。 |
| Agent | 路径不固定的多步任务。 | 步数预算、工具范围、日志和退出条件。 |

## 示例代码

下面是一个最小 LLM 调用伪代码，展示输入、上下文、模型调用和输出校验的结构：

```python
def answer(question, context):
    prompt = f"""基于资料回答问题。
资料：{context}
问题：{question}
"""
    result = call_model(prompt)
    if not result.strip():
        return {"status": "empty", "answer": None}
    return {"status": "ok", "answer": result}
```

真实系统需要把检索、权限、引用、结构化输出、日志和错误处理补齐。

## 技术细节拆解

对话式 AI 可以拆成输入、处理、输出和指标四层。这样读的时候不会停留在概念名，而是能看到它在系统里接收什么、改变什么、产出什么。

| 层次 | 具体内容 |
| --- | --- |
| 输入 | 用户问题、系统提示词、历史对话、检索片段、工具 schema。 |
| 处理 | 上下文组装、模型调用、工具调用、输出解析、权限校验。 |
| 输出 | 自然语言、JSON、引用、工具结果、拒答信息。 |
| 指标 | 准确率、引用命中率、格式成功率、工具成功率、成本、延迟。 |

## 关键参数和边界

| 参数/边界 | 说明 |
| --- | --- |
| temperature | 越高越发散，越低越稳定。 |
| max_tokens | 限制输出长度，也影响成本和截断风险。 |
| top_p | 控制采样范围，通常和 temperature 配合。 |
| schema | 结构化输出必须有 schema 和校验。 |

## 可运行检查

```python
import json

raw = '{"answer":"可以办理", "confidence":0.82}'
data = json.loads(raw)
assert "answer" in data
assert 0 <= data["confidence"] <= 1
print(data)
```

## 怎么判断学懂了

| 判断点 | 具体标准 |
| --- | --- |
| 格式稳定性 | 结构化输出能否被程序稳定解析。 |
| 证据一致性 | 回答是否能追溯到检索片段或工具结果。 |
| 权限安全 | 不同用户只能检索和调用授权范围内的资源。 |

## 常见误区和排查

| 问题 | 为什么会发生 | 怎么排查 |
| --- | --- | --- |
| 回答没有证据 | 模型自由发挥，没绑定检索片段。 | 强制引用来源并校验引用存在。 |
| 工具参数不合法 | 模型输出 JSON 不符合接口要求。 | 用 schema 校验，失败时让模型重试或拒绝执行。 |
| 权限穿透 | 检索阶段没有按用户权限过滤。 | 把用户身份加入检索条件并记录审计日志。 |

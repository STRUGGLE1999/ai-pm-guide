# 安全、合规与权限

越靠近真实业务，AI 产品越需要把安全、合规和责任边界前置。

## 知识点清单

| 知识点 | PM 要掌握的重点 | 面试可能怎么问 |
| --- | --- | --- |
| 数据权限 | 不同角色只能访问被授权的知识和工具。 | 数据权限 对 AI 产品设计有什么影响？ |
| 内容安全 | 识别敏感、违法、歧视、暴力和不适当输出。 | 内容安全 对 AI 产品设计有什么影响？ |
| 提示词注入 | 防止用户绕过规则、诱导泄露系统提示或越权调用工具。 | 提示词注入 对 AI 产品设计有什么影响？ |
| 审计日志 | 记录输入、检索、工具调用、输出和人工确认。 | 审计日志 对 AI 产品设计有什么影响？ |
| 人工确认 | 高风险动作必须由人确认后执行。 | 人工确认 对 AI 产品设计有什么影响？ |
| 可回滚设计 | 自动写入、发送、删除等动作需要撤销机制。 | 可回滚设计 对 AI 产品设计有什么影响？ |
| 隐私保护 | 最小化收集、脱敏处理和数据留存策略。 | 隐私保护 对 AI 产品设计有什么影响？ |
| 企业合规 | 关注法务、安全、采购和 IT 审批流程。 | 企业合规 对 AI 产品设计有什么影响？ |
| 模型供应商风险 | 评估可用性、数据处理方式和 SLA。 | 模型供应商风险 对 AI 产品设计有什么影响？ |
| 红队测试 | 用攻击样例提前发现安全薄弱点。 | 红队测试 对 AI 产品设计有什么影响？ |
| 免责声明 | 在医疗、法律、金融等高风险场景明确辅助定位。 | 免责声明 对 AI 产品设计有什么影响？ |
| 责任边界 | 明确 AI、用户、业务负责人和系统管理员各自负责什么。 | 责任边界 对 AI 产品设计有什么影响？ |

## 学习方法

1. 先用一句话解释概念，避免陷入纯技术细节。
2. 找一个真实产品场景，说明它如何改变用户流程。
3. 写出风险、指标和验收标准。

## 练习任务

选择 3 个知识点，各写一个「用户场景 + 产品方案 + 指标」的小卡片。
